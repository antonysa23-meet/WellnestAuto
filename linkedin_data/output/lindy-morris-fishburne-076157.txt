View company: General Sense
General Sense

 

1mo • 

Scientific ML in Python often starts with a compromise: taking richly structured data, flattening it into arrays, and manually tracking the dimensions, coordinates, and split logic somewhere else.

We built XDFlow so you don’t have to make that compromise.

XDFlow is an open-source Python framework for building ML pipelines directly on structured scientific data. It’s designed for datasets where dimensions and coordinates matter: trials, channels, time, sessions, subjects, and other experimental structure.

It lets you build composable pipelines that:

- preserve labeled dimensions and coordinates through the workflow
- validate transforms based on what they consume and produce
- split and stratify using coordinates already attached to the data
- run fold-invariant preprocessing once and refit learned steps inside training folds, keeping cross-validation leakage-safe and fast

Built on xarray, XDFlow integrates with scikit-learn, Optuna, and MLflow.

It came out of our work at General Sense on olfactory brain-computer interface research, where our ML pipelines run on structured neural recordings. But the same problem shows up across neuroscience, biosignals, imaging, climate science, geophysics, and other domains.

XDFlow is MIT-licensed and available now. The API is still evolving, so try it, break it, and tell us what does or doesn’t work for your data.

Install: pip install xdflow
Code: https://lnkd.in/eUZ9cJgJ
Docs: xdflow.readthedocs.io
